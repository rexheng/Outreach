# Borough Briefing Pack -- Design Spec

## Overview

A single-page PDF executive summary for any London borough, generated server-side via `reportlab`. One endpoint, one click, one page. Target user: borough mayor, health cabinet member, or their policy advisor.

## Content Layout (top to bottom)

### 1. Header
- "Outreach" brand name + borough name (e.g. "Hackney -- Mental Health Briefing")
- Generation date
- Subtle terracotta accent line

### 2. Headline KPIs (row of 4 boxes)
| KPI | Source | Format |
|-----|--------|--------|
| Mean CNI Score | Find matching dict in `_borough_cache` list, read `mean_lri` | X.X / 10 |
| Borough Rank | Sort all boroughs by `mean_lri` desc, find position | #N of {total} (1 = highest need). Use `len(_borough_cache)` -- do not hardcode 33 |
| Critical Neighbourhoods | `matched_borough["critical_count"]` | N areas |
| COVID Impact | `matched_borough["samhi_covid_change"]` | +X.XX arrow up/down |

**Borough lookup**: `_borough_cache` is a `list[dict]`. Find the matching entry with:
```python
borough_data = next((b for b in _borough_cache if b["borough"].lower() == borough_name.lower()), None)
```

### 3. Top 5 Priority Neighbourhoods (table)
Filter `_gdf_cache` by `Local Authority District name (2019)` == borough (case-insensitive), sort by `lri_score` desc, take top 5.

| Column | Source |
|--------|--------|
| Neighbourhood | `lsoa_name`, fallback to `lsoa_code` if null |
| CNI Score | `lri_score` |
| Tier | `risk_tier` |
| Primary Driver | Row-level comparison: if `pillar_socioeconomic > pillar_demographic` then "Socioeconomic deprivation", else "Demographic vulnerability" |

### 4. Key Drivers Narrative (2-3 sentences, auto-generated)
Template-based, no LLM. Deterministic, fast, zero API cost.

**Logic (row-level pillar comparison):**
```python
borough_gdf = _gdf_cache[_gdf_cache[BOROUGH_COL].str.lower() == borough_name.lower()]
critical_lsoas = borough_gdf[borough_gdf["risk_tier"] == "Critical"]
socio_dominant = (critical_lsoas["pillar_socioeconomic"] > critical_lsoas["pillar_demographic"]).sum()
demo_dominant = len(critical_lsoas) - socio_dominant
```

**Narrative paths:**
- **Has critical LSOAs**: "{Dominant pillar} is the primary driver in {borough}, with {N} of {total} critical-need neighbourhoods driven by {pillar label}. {Notable indicator sentence if above London avg}."
- **Zero critical LSOAs (fallback)**: "{Borough} has no critical-need neighbourhoods. The mean CNI score is {score}, ranking {rank} of {total} boroughs. {Dominant pillar} accounts for the larger share of composite need."

Example output: "Socioeconomic deprivation is the primary driver in Hackney, with 8 of 12 critical-need neighbourhoods driven by deprivation indicators. Antidepressant prescribing is 15% above the London average."

### 5. Borough vs London Comparison (mini table, 5 rows)

**London averages must be LSOA-weighted** (computed from `_gdf_cache`, not mean-of-borough-means) to avoid size bias:
```python
london_cni_avg = _gdf_cache["lri_score"].mean()
london_samhi_avg = _gdf_cache["samhi_index_2022"].mean()
# etc -- use _gdf_cache column-level means, not _borough_cache
```

| Indicator | Borough Avg | London Avg | Difference |
|-----------|------------|------------|------------|
| CNI Score | `mean_lri` | `_gdf_cache["lri_score"].mean()` | +/- |
| SAMHI Index (2022) | `samhi_mean` | `_gdf_cache["samhi_index_2022"].mean()` | +/- |
| Antidepressant Rate | `antidep_mean` | `_gdf_cache["antidep_rate_2022"].mean()` | +/- |
| Bad/Very Bad Health % | `bad_health_mean` | `_gdf_cache["health_bad_or_very_bad_pct"].mean()` | +/- |
| IMD Score | `imd_mean` | `_gdf_cache["imd_score"].mean()` | +/- |

NaN values excluded from averages (pandas default). Footer notes this.

### 6. Footer
- "Data: IMD 2019, SAMHI 2022, Census 2021, ONS LSOA boundaries 2021. NaN values excluded from averages."
- "Generated by Outreach -- The Geography of Wellbeing"
- Dashboard URL

## Technical Design

### New Files
- `app/api/briefing.py` -- endpoint + PDF builder

### Endpoint
```
GET /api/briefing/{borough_name}
```
- Use `APIRouter(prefix="/api")` with `@router.get("/briefing/{borough_name}")` to match existing `routes.py` pattern
- Use **sync `def`**, not `async def` -- reportlab PDF generation is CPU-bound and would block the event loop
- Returns `application/pdf` with `Content-Disposition: attachment; filename="Outreach-Briefing-{sanitized}.pdf"` where spaces become hyphens
- 404 via `HTTPException(status_code=404, detail="Borough not found: {borough_name}")` if no match
- Guard: call `load_and_prepare()` at handler top to ensure data is loaded (matches pattern in `loader.py` accessor functions)
- FastAPI handles URL-decoding of path params automatically (e.g. `Hammersmith%20and%20Fulham` -> `Hammersmith and Fulham`)

### PDF Generation
- Library: `reportlab` (add to `requirements.txt` -- pure Python, no system-level C deps, Railway-safe)
- Page size: A4 landscape (more room for KPI row + table)
- Fonts: Helvetica (reportlab built-in) -- no need to embed custom fonts for a data PDF
- Colour accents: terracotta `#B5725A` for headers, `#6B4A3A` for critical tier, `#FAF7F3` for background fills

### Data Access
All data already in memory via `loader._gdf_cache` and `loader._borough_cache`:
- Borough stats: `loader._borough_cache` (list of dicts, one per borough)
- LSOA-level: filter `loader._gdf_cache` by `Local Authority District name (2019)`
- London averages: compute from `_gdf_cache` column-level means (LSOA-weighted, not mean-of-borough-means)

### Frontend Trigger
- Button placed inside the borough filter `.panel-section` in `#left-panel`, after `#borough-select`
- Hidden by default, shown via JS when `#borough-select` has a non-empty value
- Label: "Download Briefing"
- On click: `window.location.href = '/api/briefing/' + encodeURIComponent(selectedBorough)`
- Styled as secondary action (outline button in terracotta)

### Registration
- Import and register router in `app/main.py` via `app.include_router()`
- Add `reportlab` to `requirements.txt`

## Out of Scope
- Multi-page reports (single-page by design)
- Charts/graphs in PDF (table + text for v1; charts can come later)
- LLM-generated narratives
- Custom section toggles
- Ward-level aggregation
- Standard-deviation outlier detection in narrative (defer to v2)
